<dl> <dt>

<span id="_rpc_rpc_object_glos"></span><span id="_RPC_RPC_OBJECT_GLOS"></span>**Objeto RPC**
</dt> <dd>

Instancias de servidor u otros recursos, como dispositivos, bases de datos y colas, que funcionan y administran las aplicaciones de servidor RPC. Cada objeto se identifica de forma única mediante uno o varios UUID de objeto.

</dd> <dt>

<span id="_rpc_rpcss_glos"></span><span id="_RPC_RPCSS_GLOS"></span>**Subsistema RPC (RPCSS)**
</dt> <dd>

Windows que incluye una variedad de servicios RPC y OLE, incluidos el asignador de puntos de conexión, el Administrador de control de servicios OLE (SCM) y el solucionador de objetos COM. No confunda esto con el paquete de asignador de memoria específico de RPC, RpcSs.

</dd> <dt>

<span id="_rpc_reference_pointer_glos"></span><span id="_RPC_REFERENCE_POINTER_GLOS"></span>**puntero de referencia**
</dt> <dd>

Tipo de puntero más sencillo. Un puntero de referencia siempre apunta al almacenamiento válido y ese almacenamiento no cambia (aunque el contenido puede cambiar). No se puede crear un alias para un puntero de referencia. El \[ [atributo ref](/windows/desktop/Midl/ref) \] designa un puntero de referencia. Vea también [*puntero único y*](u-glos.md) puntero [*completo.*](f-glos.md)

</dd> </dl>
