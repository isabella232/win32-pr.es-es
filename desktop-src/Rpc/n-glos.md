# <a name="n-rpc"></a>N (RPC)

<dl> <dt>

<span id="_rpc_name_service_glos"></span><span id="_RPC_NAME_SERVICE_GLOS"></span>**servicio de nombres**
</dt> <dd>

Servicio que asigna nombres a objetos y almacena los pares nombre-objeto en una base de datos. Por ejemplo, el servicio de nombres RPC asigna un nombre lógico [*a*](b-glos.md) un identificador de enlace para que las aplicaciones cliente puedan hacer referencia a ese nombre lógico, en lugar de a una secuencia de protocolo y una dirección de red. Vea también name service–interface daemon [(nsid)](/windows), Client Directory Service [*(CDS)*](c-glos.md), [*Locator*](l-glos.md).

</dd> <dt>

<span id="_rpc_nca_glos"></span><span id="_RPC_NCA_GLOS"></span>**Arquitectura de informática de red (NCA)**
</dt> <dd>

Colección de directrices para la computación distribuida. Los protocolos de comunicación RPC siguen estas directrices.

</dd> <dt>

<span id="_rpc_nsi_glos"></span><span id="_RPC_NSI_GLOS"></span>**Name Service Independent (NSI)**
</dt> <dd>

Estándar para funciones de API que permite que una aplicación distribuida acceda a los elementos de base de datos rpc name-service a través de varios proveedores de servicios de nombres, como OSF-DCE Servicio de directorio de celdas o Microsoft Locator. Consulte también [–interface daemon](/windows) (nsid).

</dd> <dt>

<span id="_rpc_nsid_glos"></span><span id="_RPC_NSID_GLOS"></span>**name service–interface daemon (nsid)**
</dt> <dd>

Servicio que proporciona una interfaz entre Microsoft [*Locator*](l-glos.md) y OSF-DCE Servicio de directorio de celdas de servicio de nombres para funciones de servicio de nombre RPC.

</dd> <dt>

<span id="_rpc_named_pipe_glos"></span><span id="_RPC_NAMED_PIPE_GLOS"></span>**canalización con nombre**
</dt> <dd>

Protocolo orientado a la conexión, basado en bloques de mensajes de servidor (SMB) y [NetBIOS](/windows), que se usa para la comunicación entre un proceso de servidor y uno o varios procesos de cliente.

</dd> <dt>

<span id="_rpc_netbeui_glos"></span><span id="_RPC_NETBEUI_GLOS"></span>**NetBIOS Extended Interfaz de usuario (NetBEUI)**
</dt> <dd>

Protocolo de transporte nativo de LAN Manager y controlador de dispositivo de red. Consulte también [NetBIOS.](/windows)

</dd> <dt>

<span id="_rpc_netbios_glos"></span><span id="_RPC_NETBIOS_GLOS"></span>**Sistema de entrada y salida básico de red (NetBIOS)**
</dt> <dd>

Interfaz de software entre el sistema operativo Microsoft MS-DOS, el bus de E/S y una red de área local.

</dd> <dt>

<span id="_rpc_ndr_glos"></span><span id="_RPC_NDR_GLOS"></span>**Representación de datos de red (BAND)**
</dt> <dd>

Formato estándar usado durante la transmisión de red que es independiente del formato de tipo de datos en cualquier arquitectura de equipo determinada. Los datos transmitidos incluyen información que especifica su formato JPEG.

</dd> <dt>

<span id="_rpc_network_address_glos"></span><span id="_RPC_NETWORK_ADDRESS_GLOS"></span>**dirección de red**
</dt> <dd>

Dirección que identifica un servidor en una red.

</dd> <dt>

<span id="_rpc_nonencapsulated_union_glos"></span><span id="_RPC_NONENCAPSULATED_UNION_GLOS"></span>**unión nonencapsulated**
</dt> <dd>

[*Unión discriminada que*](d-glos.md) es menos restrictiva que una unión encapsulada en que el discriminador y la unión no están estrechamente enlazados. Si la unión es un parámetro, el discriminante es otro parámetro; Si la unión es un campo de estructura, el discriminador es otro campo de estructura. El modificador de palabras clave IDL \[ [**\_ es y**](/windows/desktop/Midl/switch-is) el tipo \] \[ [**\_ switch**](/windows/desktop/Midl/switch-type) \] identifica el discriminante y su tipo. Consulte también [*unión encapsulada*](e-glos.md).

</dd> <dt>

<span id="_rpc_nonidempotent_glos"></span><span id="_RPC_NONIDEMPOTENT_GLOS"></span>**noidempotente**
</dt> <dd>

Indicador de que una llamada a procedimiento remoto no se puede ejecutar más de una vez porque devolverá un valor diferente o cambiará un estado.

</dd> </dl>
