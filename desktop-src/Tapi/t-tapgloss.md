<dl> <dt>

<span id="tapi2.t1_e1_tapgloss"></span><span id="TAPI2.T1_E1_TAPGLOSS"></span>**T1/E1**
</dt> <dd>

Un vínculo de transmisión digital con una capacidad de 1544 Mbps. En Estados Unidos, este vínculo se denomina *T1.* En la Unión Europea, se denomina *E1.* Vea [*megabits por segundo (Mbps).*](m-tapgloss.md)

</dd> <dt>

<span id="tapi2.tapi_tapgloss"></span><span id="TAPI2.TAPI_TAPGLOSS"></span>**Tapi**
</dt> <dd>

Vea *Interfaz de programación de aplicaciones de telefonía (TAPI).*

</dd> <dt>

<span id="tapi2.tapisrv_tapgloss"></span><span id="TAPI2.TAPISRV_TAPGLOSS"></span>**tapisrv**
</dt> <dd>

Una aplicación de servicio de soporte técnico que proporciona un contexto de ejecución para que la biblioteca de vínculos dinámicos TAPI lleve a cabo tareas como asignar memoria o acceder a estructuras de datos que se han intercambiado al disco.

</dd> <dt>

<span id="tapi2.tdd_tapgloss"></span><span id="TAPI2.TDD_TAPGLOSS"></span>**Tdd**
</dt> <dd>

Consulte *Telecommunications Devices for the Deaf (TDD) [Dispositivos de telecomunicaciones para el dispositivo de telecomunicaciones (TDD)].*

</dd> <dt>

<span id="tapi2.telecommucations_devices_for_the_deaf_tdd__tapgloss"></span><span id="TAPI2.TELECOMMUCATIONS_DEVICES_FOR_THE_DEAF_TDD__TAPGLOSS"></span>**Telecommucations Devices for the Deaf (TDD)**
</dt> <dd>

Un dispositivo de datos para la transmisión de señales codificadas. Normalmente, el dispositivo es un terminal de equipo simple que consta de un teclado, un acoplador acústico y una pantalla, y se ejecuta a 300 baudios.

</dd> <dt>

<span id="tapi2.telephony_tapgloss"></span><span id="TAPI2.TELEPHONY_TAPGLOSS"></span>**Telefonía**
</dt> <dd>

La ciencia de la transmisión de señales de voz, datos, vídeo o imágenes a lo largo de una distancia mediante tecnología que integra los equipos con la red telefónica.

</dd> <dt>

<span id="tapi2.telephony_application_programming_interface_tapi__tapgloss"></span><span id="TAPI2.TELEPHONY_APPLICATION_PROGRAMMING_INTERFACE_TAPI__TAPGLOSS"></span>**Interfaz de programación de aplicaciones de telefonía (TAPI)**
</dt> <dd>

Conjunto de funciones que permite programar dispositivos basados en línea telefónica de forma independiente del dispositivo, lo que da telefonía personal a los usuarios. TAPI admite la transmisión de voz y datos, permite una variedad de dispositivos terminales y admite tipos de conexión complejos y técnicas de administración de llamadas, como llamadas de conferencia, espera de llamadas y correo de voz. TAPI permite controlar todos los elementos del uso del teléfono,desde la simple llamada telefónica a correo electrónico internacional, dentro de las aplicaciones desarrolladas para Microsoft Windows. Consulte Introducción [a la telefonía de Microsoft.](./microsoft-telephony-overview.md)

</dd> <dt>

<span id="tapi2.telephony_service_provider_interface_tspi__tapgloss"></span><span id="TAPI2.TELEPHONY_SERVICE_PROVIDER_INTERFACE_TSPI__TAPGLOSS"></span>**Interfaz del proveedor de servicios de telefonía (TSPI)**
</dt> <dd>

Herramienta para crear proveedores de servicios para microsoft Windows sistemas operativos. TSPI define cómo la red comparte información con Windows Telefonía, que a su vez se pone en contacto con la API, que se pone en contacto con las Windows telefonía. Para obtener más información, vea [Interfaz del proveedor de servicios de telefonía (TSPI).](./telephony-service-provider-interface-tspi-.md)

</dd> <dt>

<span id="tapi2.third_party_call_control_tapgloss"></span><span id="TAPI2.THIRD_PARTY_CALL_CONTROL_TAPGLOSS"></span>**control de llamadas de terceros**
</dt> <dd>

Modelo de control de llamadas de entidad que permite a una aplicación establecer o responder a una llamada entre dos partes cualquiera.

</dd> <dt>

<span id="tapi2.time_to_live_ttl__tapgloss"></span><span id="TAPI2.TIME_TO_LIVE_TTL__TAPGLOSS"></span>**período de vida (TTL)**
</dt> <dd>

Valor del intervalo de 0 a 255 que define el ámbito dentro del cual se deben enviar paquetes de multidifusión a través de una red mediante el Protocolo de Internet (IP). El ámbito se define en términos de lo local o remoto que es el destino de un paquete. Cada enrutador disminuye el TTL en uno. Cuando el valor alcanza un límite inferior predefinido, el enrutador lanza el paquete. Los requisitos actuales de la red troncal de multidifusión (MCCI), disponibles en el sitio de ftp://ftp.isi.edu/mbone/faq.txt, definen los siguientes ámbitos estándar: red local, 1; sitio local, 15; región, 63; world, 127. Otras configuraciones pueden tener significado local; Por ejemplo, 31 podría indicar todos los sitios dentro de una organización determinada.

</dd> <dt>

<span id="tapi2.tspi_tapgloss"></span><span id="TAPI2.TSPI_TAPGLOSS"></span>**TSPI**
</dt> <dd>

</dd> <dt>

<span id="tapi2.ttl_tapgloss"></span><span id="TAPI2.TTL_TAPGLOSS"></span>**Ttl**

Vea *período de vida (TTL).*

</dd> <dt>

<span id="tapi2.twisted_pair_tapgloss"></span><span id="TAPI2.TWISTED_PAIR_TAPGLOSS"></span>**par de torsión**
</dt> <dd>

Cableado telefónico normal. Cada cable telefónico es realmente un par de cables.

</dd> </dl>
