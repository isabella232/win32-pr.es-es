# <a name="terminal-object"></a><span data-ttu-id="66f6b-103">Objeto terminal</span><span class="sxs-lookup"><span data-stu-id="66f6b-103">Terminal Object</span></span>

<span data-ttu-id="66f6b-104">En la versión 3,0 de TAPI y versiones posteriores, el modelo de objetos de TAPI utiliza objetos de terminal para representar el origen o el receptor de un flujo multimedia asociado a una sesión de llamada o de comunicaciones.</span><span class="sxs-lookup"><span data-stu-id="66f6b-104">In TAPI version 3.0 and later, the TAPI object model uses terminal objects to represent the source or sink of a media stream associated with a call or communications session.</span></span> <span data-ttu-id="66f6b-105">Este modelo de objetos permite a una aplicación especificar, en un nivel detallado, cómo se procesan los elementos multimedia en una llamada.</span><span class="sxs-lookup"><span data-stu-id="66f6b-105">This object model enables an application to specify, at a detailed level, how media are processed on a call.</span></span> <span data-ttu-id="66f6b-106">Este modelo también permite seleccionar varios terminales simultáneamente, por lo que, por ejemplo, se puede enviar una llamada a un altavoz de audio y grabarse simultáneamente.</span><span class="sxs-lookup"><span data-stu-id="66f6b-106">This model also enables multiple terminals to be selected simultaneously, so for example, a call can be output to an audio speaker and recorded simultaneously.</span></span>

<span data-ttu-id="66f6b-107">El objeto terminal representa un origen o un representador, como un micrófono o un altavoz.</span><span class="sxs-lookup"><span data-stu-id="66f6b-107">The Terminal object represents a source or renderer, such as a microphone or a speaker.</span></span> <span data-ttu-id="66f6b-108">Una aplicación elige entre los terminales disponibles en función de la dirección del medio y tipo o tipos implicados en una sesión de comunicaciones.</span><span class="sxs-lookup"><span data-stu-id="66f6b-108">An application chooses among available terminals based on the media direction and type or types involved in a communications session.</span></span> <span data-ttu-id="66f6b-109">Cada flujo de medios asociado se selecciona en el terminal adecuado para iniciar el streaming.</span><span class="sxs-lookup"><span data-stu-id="66f6b-109">Each associated media stream is then selected onto the appropriate terminal in order to start streaming.</span></span>

<span data-ttu-id="66f6b-110">Los terminales se implementan normalmente mediante un proveedor de servicios multimedia (MSP) y los objetos de terminal no estarán disponibles si no hay ningún MSP asociado a una sesión de comunicaciones.</span><span class="sxs-lookup"><span data-stu-id="66f6b-110">Terminals are normally implemented by a media service provider (MSP) and terminal objects will not be available if there is no MSP associated with a communications session.</span></span> <span data-ttu-id="66f6b-111">Una excepción es que con Windows 2000 SP1 y versiones posteriores, una aplicación puede implementar una forma de [terminal conectable](pluggable-terminals.md).</span><span class="sxs-lookup"><span data-stu-id="66f6b-111">One exception is that with Windows 2000 SP1 and later, an application can implement a form of [pluggable terminal](pluggable-terminals.md).</span></span> <span data-ttu-id="66f6b-112">Esto permite que un servidor de conferencia cree terminales de puente de modo que se puedan agregar a los clientes de la multidifusión de los servidores SDP/IP de Windows 2000 SP1 o de multidifusión.</span><span class="sxs-lookup"><span data-stu-id="66f6b-112">This enables a conference server to create bridging terminals so that non-Windows 2000 SP1 or non-multicast H323 clients can be added to TAPI 3 multi-party SDP/IP multicast conferences.</span></span>

<span data-ttu-id="66f6b-113">Cada terminal pertenece a una [clase de terminal](terminal-class.md).</span><span class="sxs-lookup"><span data-stu-id="66f6b-113">Each terminal belongs to a [terminal class](terminal-class.md).</span></span> <span data-ttu-id="66f6b-114">Una clase de terminal representa un conjunto de características de origen o representación.</span><span class="sxs-lookup"><span data-stu-id="66f6b-114">A terminal class represents a set of source or render features.</span></span> <span data-ttu-id="66f6b-115">Por ejemplo, un terminal que se asigna a un conjunto de altavoces de audio se identificaría como CLSID \_ SpeakersTerminal y se espera que el proveedor de servicios implemente el control de volumen.</span><span class="sxs-lookup"><span data-stu-id="66f6b-115">For example, a terminal that maps to a set of audio speakers would be identified as CLSID\_SpeakersTerminal, and the service provider would be expected to implement volume control.</span></span> <span data-ttu-id="66f6b-116">TAPI 3 define un conjunto de clases de terminal, un MSP puede definir clases adicionales y una aplicación puede registrar nuevas clases de terminal.</span><span class="sxs-lookup"><span data-stu-id="66f6b-116">TAPI 3 defines a set of terminal classes, an MSP can define additional classes, and an application can register new terminal classes.</span></span> <span data-ttu-id="66f6b-117">A cada clase de terminal se le asigna un identificador único global (GUID).</span><span class="sxs-lookup"><span data-stu-id="66f6b-117">Each terminal class is assigned a globally unique identifier (GUID).</span></span>

<span data-ttu-id="66f6b-118">Desde el punto de vista de una aplicación, un terminal se describe mediante su [tipo de terminal](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_type) y [Dirección](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_direction).</span><span class="sxs-lookup"><span data-stu-id="66f6b-118">From the point of view of an application, a terminal is described by its [terminal type](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_type) and [direction](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_direction).</span></span> <span data-ttu-id="66f6b-119">El tipo puede ser estático o dinámico.</span><span class="sxs-lookup"><span data-stu-id="66f6b-119">The type may be either static or dynamic.</span></span> <span data-ttu-id="66f6b-120">Un terminal estático se asigna a hardware, como un teléfono o micrófono.</span><span class="sxs-lookup"><span data-stu-id="66f6b-120">A static terminal maps to hardware, such as a telephone or microphone.</span></span> <span data-ttu-id="66f6b-121">Un terminal dinámico se asigna a un objeto transitorio, como un archivo o una ventana de vídeo.</span><span class="sxs-lookup"><span data-stu-id="66f6b-121">A dynamic terminal maps to a transient object, such as a file or a video window.</span></span> <span data-ttu-id="66f6b-122">Direction describe si un terminal determinado es un origen o un representador.</span><span class="sxs-lookup"><span data-stu-id="66f6b-122">Direction describes whether a given terminal is a source or a renderer.</span></span>
