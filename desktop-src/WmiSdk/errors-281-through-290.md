## <a name="fatal-error-282"></a>Error irresal 282

<dl> <dt>

<span id="_282__Fatal_____fileName___line____Syntax_error._Last_token_read_is__token__"></span><span id="_282__fatal_____filename___line____syntax_error._last_token_read_is__token__"></span><span id="_282__FATAL_____FILENAME___LINE____SYNTAX_ERROR._LAST_TOKEN_READ_IS__TOKEN__"></span>**<282, Error>: " &lt; fileName &gt; :<line \#>: Error de sintaxis. La última lectura del token es &lt; token &gt; "**
</dt> <dd>

Este error se produce cuando el compilador sabe que se ha producido un error de sintaxis, pero no puede proporcionar información más específica sobre el error de sintaxis.

</dd> </dl>

## <a name="fatal-error-283"></a>Error irresal 283

<dl> <dt>

<span id="_283__Fatal_____fileName___line_____Unrecognized_character__character__in_input_"></span><span id="_283__fatal_____filename___line_____unrecognized_character__character__in_input_"></span><span id="_283__FATAL_____FILENAME___LINE_____UNRECOGNIZED_CHARACTER__CHARACTER__IN_INPUT_"></span>**<283, Fatal>: " &lt; fileName &gt; :<line \# \#>: Unrecognized character &lt; &gt; character in input"**
</dt> <dd>

Hay un carácter no reconocido en la entrada. Normalmente, se omiten los caracteres no reconocidos y este mensaje de error notifica el primer carácter no reconocido.

</dd> </dl>
