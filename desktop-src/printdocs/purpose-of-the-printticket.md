# <a name="purpose-of-the-printticket"></a>Propósito de PrintTicket

Este tema no es actual. Para obtener la información más reciente, vea [La especificación de esquema de impresión](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

El propósito principal de PrintTicket es expresar configuraciones de dispositivo. A diferencia del documento PrintCapabilities, que define todas las configuraciones posibles que el dispositivo puede asumir (el espacio de configuración del dispositivo), el propósito de PrintTicket es definir una única configuración específica que se aplica al objeto de trabajo (en este caso, la página) que se está procesando. Para promover la portabilidad y la coherencia de las configuraciones, tanto las tecnologías PrintCapabilities como PrintTicket usan las palabras clave y la jerarquía específicas que se presentan en la sección Palabras clave de esquema de impresión. Además, el marco de [esquema](print-schema-framework.md) de impresión define las construcciones básicas de ambas tecnologías, para promover una comunicación inequívoca, abierta y extensible del formato de trabajo y PrintCapabilities.
