# <a name="terms-used-in-the-print-schema"></a>Términos usados en el esquema de impresión

Este tema no es actual. Para obtener la información más reciente, vea [La especificación de esquema de impresión](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

La lista siguiente contiene términos que son específicos del esquema de impresión y las tecnologías relacionadas.



| Término                                                                                                                                                                 | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>nombre del atributo<br/>                                                                          | Un nombre de atributo es el token del lado izquierdo del signo igual en un atributo XML estándar. La representación estándar de un atributo XML es *el nombre de atributo*=" valor de *atributo*". Aunque es similar, el término "nombre de atributo" no debe confundirse con el atributo name definido en el marco de esquema de impresión. El atributo name definido en el marco de esquema de impresión es una instancia específica de un nombre de atributo. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>Candidato<br/>                                                                                         | Opción. Cualquier opción que pertenezca a la lista que se está evaluando durante el proceso de puntuación. Consulte los términos, opción de puntuación o asignación y opción de referencia.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Configuración<br/>                                             | Describe uno de los estados en los que un dispositivo puede asumir o colocarse. El dispositivo realiza su tarea de forma diferente en cada estado. Hay una diferencia distinta e identificable entre el comportamiento del dispositivo en cada uno de estos estados. Por ejemplo, en un estado, el medio se saca de una bandeja y termina en otra. Un cambio en una de estas bandejas o en ambas determina un estado diferente.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Restricción<br/>                                                         | Información necesaria para determinar la validez de una configuración. Una restricción suele tener el formato: FeatureA establecido en Option1 no es compatible con FeatureB establecido en Option2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>elementos correspondientes<br/>                                                  | Dos elementos que tienen la misma ubicación en relación con un par especificado de elementos de origen. Los elementos de origen suelen ser dos elementos Option que se comparan.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>Predeterminado<br/>                                                                                               | Opción. La opción designada por el esquema de impresión o el proveedor PrintCapabilities como opción que se asigna a una característica cuando PrintTicket no contiene una configuración explícita para esa característica. Cada característica tiene una opción predeterminada designada.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>atributo de configuración de dispositivo<br/>                          | Atributo de dispositivo que contribuye a la definición de la configuración del dispositivo. Un atributo de configuración de dispositivo se representa en el marco de esquema de impresión mediante construcciones de característica/opción o parámetro. Tenga en cuenta que el atributo utilizado en este contexto no tiene ninguna relación con los atributos XML ni con los elementos Attribute definidos por el marco de esquema de impresión.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>tipo de elemento<br/>                                                                                | El tipo de cada elemento XML definido en el marco de esquema de impresión se identifica mediante su etiqueta de elemento XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>elementos en común<br/>                                                              | Vea los elementos correspondientes.<br/> Característica. Construcción definida por el marco de trabajo de esquema de impresión que representa un atributo de configuración de dispositivo específico. Cada instancia de característica representa un atributo de dispositivo. Una instancia de característica se define mediante la definición explícita de cada uno de sus estados.<br/> Opción de identidad. Opción que no tiene ningún efecto observable cuando se selecciona; en efecto, un "no-op". El resultado es siempre el mismo, independientemente del número de veces que se seleccione esta opción.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>nombre de instancia<br/>                                                                             | Atributo XML que identifica la instancia específica de un tipo de elemento. Vea name attribute (Atributo de nombre).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>cliente de interfaz<br/>                                                                    | Entidad que consume documentos PrintCapabilities y que construye y consume PrintTickets. Entre los ejemplos comunes de clientes de interfaz se incluyen los módulos de interfaz de usuario (UI) y los módulos de representación.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>proveedor de interfaz<br/>                                                              | Entidad que implementa y expone la funcionalidad mediante una interfaz . En el esquema de impresión, esto equivale a crear documentos PrintCapabilities y validar PrintTickets. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>ubicación de un elemento<br/>                                                  | Posición de una instancia determinada de un elemento dentro de la jerarquía de elementos. La ubicación se define tanto por el elemento concreto como por todos los antecesores de ese elemento. En otras palabras, location es el equivalente de una ruta de acceso completa en un sistema de archivos. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>atributo name<br/>                                                                          | Designación dada al atributo XML con un nombre de atributo de "name". Este atributo aparece en la mayoría de los tipos de elemento definidos en el marco de esquema de impresión. Identifica y define instancias de cada tipo de elemento. Los dos términos siguientes hacen referencia a los atributos de nombre: la regla de unidad del mismo nivel y la convención de espacio de nombres.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Convención de espacio de nombres<br/>                 | Indica que los atributos de nombre definidos por el esquema, así como cualquier objeto definido por el esquema, existen en un espacio de nombres dedicado, el espacio de nombres público y que los atributos de nombre definidos de forma privada deben existir en un espacio de nombres diferente, uno que se pueda identificar como perteneciente a la entidad de definición.<br/> Opción. Construcción del marco de esquema de impresión que se usa para representar los estados en los que se puede asumir o colocar cada característica. Por ejemplo, si la característica representa el tamaño del medio, los valores de opción típicos podrían ser Letter, Legal, A4, entre otros.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Parámetro<br/>                                                             | El esquema de impresión admite la capacidad de describir datos con parámetros para la entrada del usuario. Hay tres tipos de elementos de Marco de esquema de impresión que desempeñan roles en la entrada paramétrica: ParameterDef, ParameterInit y ParameterRef. Un elemento ParameterDef caracteriza un parámetro (su intervalo de valores, el tipo de sus datos y otras características). Un elemento ParameterInit se usa para inicializar un parámetro. Un elemento ParameterRef se usa para hacer referencia a una instancia determinada de un elemento ParameterInit.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>Parametrizados<br/>                                                                             | Opción. Instancia de Option que contiene al menos un elemento ScoredProperty que, a su vez, contiene un elemento ParameterRef en lugar de un elemento Value.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>perfectamente igual a<br/>                                                              | Opciones. Se considera que dos instancias de Option coinciden perfectamente si, para cada elemento ScoredProperty de una instancia de Option, hay una instancia scoredProperty correspondiente en la otra instancia de Option y ambos elementos Value de las instancias scoredProperty correspondientes son idénticos.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Esquema PrintCapabilities<br/> | Esquema que define la sintaxis y la semántica utilizadas por un documento PrintCapabilities. El esquema PrintCapabilities y el esquema PrintTicket son subconjuntos del esquema de impresión. Consulte también Marco de esquema de impresión.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>Esquema printticket<br/>                         | Esquema que define la sintaxis y la semántica utilizadas por PrintTicket. El esquema PrintTicket y el esquema PrintCapabilities son subconjuntos del esquema de impresión. Consulte también Marco de esquema de impresión.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Marco de esquema de impresión<br/>         | Esquema que define la sintaxis y la semántica utilizadas por las tecnologías de esquema de impresión. Por ejemplo, Print Schema Framework define una colección de tipos de elementos y sus usos, pero no enumera todas las instancias de elemento.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Palabras clave de esquema de impresión<br/>             | Esquema que define instancias individuales de los elementos Property, Feature y Option que se usan normalmente para describir los dispositivos y el formato del trabajo. Este esquema define un lenguaje común que hace portables los documentos PrintCapabilities y PrintTickets. También se conoce como esquema público.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>definidos de forma privada<br/>                                                                 | Instancias de elementos que aparecen en un documento PrintCapabilities o PrintTicket que no están definidos explícitamente en las palabras clave de esquema de impresión. Estas instancias de elemento deben calificarse explícitamente con un espacio de nombres privado por proveedor.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>Referencia<br/>                                                                                         | Opción. Opción con la que se compara cada opción candidata. Normalmente, una opción de referencia se origina en PrintTicket. Consulte los términos, la opción de puntuación o asignación y la opción candidata.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>nivel raíz<br/>                                                                                      | El nivel raíz es la base de la estructura jerárquica de Print Schema Framework. Un elemento de nivel raíz se define como un elemento secundario de un elemento PrintCapabilities o un elemento PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>prefijo de ámbito<br/>                                                                          | Etiqueta textual anexada previamente a una palabra clave de esquema para proporcionar un ámbito contextual. Los prefijos de ámbito permitidos del esquema de impresión son Job, Doc y Page. Por ejemplo, el prefijo de ámbito de JobInputBin es "Job".<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Construcción de Marco de esquema de impresión que se usa para representar las calidades que distinguen o diferencian una instancia de Option de otra. Un ejemplo es la diferencia entre el papel de tamaño letra y el papel de tamaño legal.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>puntuación o asignación<br/>                                                              | Opción. El proceso de seleccionar de una lista la instancia de opción (candidata) que mejor coincida con una instancia de opción determinada (referencia). Los términos estrechamente relacionados son opción candidata y opción de referencia. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Regla de unidad del mismo nivel<br/>     | Indica que todos los elementos del mismo tipo de elemento deben tener atributos de nombre únicos. Algunos tipos de elementos están exentos de esta regla. Estas excepciones se definen en el marco de esquema de impresión.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>de un solo valor frente a multivalor<br/>                    | Propiedad. Una propiedad cuyo valor es invariable (no depende de la configuración) se considera de un solo valor. Esta propiedad solo tiene un valor único en cualquier configuración posible. Una propiedad que depende de la configuración se considera multivalor.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>sufijo/sufijos<br/>                                                                       | Palabra clave de esquema que representa el nombre de la característica o del parámetro. Por ejemplo, el sufijo de JobInputBin es "InputBin".<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Validación<br/>                                                         | El proceso de comprobar que printticket se ajusta al esquema y representa una configuración que un dispositivo específico puede ejecutar.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Indefinido\_<br/>                                                   | Los datos específicos que se representan aquí son un marcador de posición para el controlador, dispositivo o aplicación específicos que se va a generar. No transmite que los datos deben ser una cadena.<br/>                                                                                                                                                                                                                                                                                                                                                            |
