# <a name="object-state-constants"></a>Constantes de estado de objeto

En este tema se describen los valores constantes utilizados para describir el estado de los objetos en una interfaz de usuario de la aplicación. Las constantes de estado se definen en oleacc.h.

Un objeto está asociado a uno o varios de estos valores de estado en cualquier momento. No se usan las siguientes constantes de estado de objeto: STATE \_ SYSTEM \_ ALERT \_ HIGH, STATE SYSTEM ALERT \_ \_ \_ MEDIUM, STATE SYSTEM ALERT \_ LOW y STATE SYSTEM \_ \_ \_ \_ FLOATING.

Los clientes recuperan el estado de un objeto mediante una llamada a [**IAccessible::get \_ accState**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accstate), que devuelve un entero que es una combinación de las siguientes marcas de bits. Los clientes [**llaman a GetStateText**](/windows/desktop/api/Oleacc/nf-oleacc-getstatetexta) con el valor de estado para recuperar una cadena localizada que describe el estado del objeto.

Cuando cambia el estado de un objeto, los servidores deben llamar [**a NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) con la [**constante de evento EVENT OBJECT \_ \_ STATECHANGE.**](event-constants.md) Sin embargo, los objetos con las constantes de estado de objeto STATE SYSTEM INVISIBLE, STATE SYSTEM FOCUSED y STATE SYSTEM SELECTED tienen \_ \_ sus \_ \_ \_ \_ propias constantes de evento. Para estos objetos, no use **EVENT \_ OBJECT \_ STATECHANGE**. En su lugar, use la constante de evento individual.
