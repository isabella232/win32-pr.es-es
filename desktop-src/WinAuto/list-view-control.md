# <a name="list-view-control-msaa-ui-element-reference"></a>List-View control (Referencia de elementos de la interfaz de usuario de MSAA)

> [!Note]  
> En este tema se describen **los objetos de control List-View** para fines de referencia de elementos de interfaz de usuario de MSAA. Aquí no se describe cómo crear objetos **de control List-View** en varios marcos de interfaz de usuario. Consulte la documentación de referencia de API para el marco de interfaz de usuario que está usando.

 

Un control de vista de lista muestra una colección de elementos, cada uno de los que consta de un icono y una etiqueta, y proporciona varias maneras de mostrar y organizar los elementos. Por ejemplo, se muestra información adicional sobre cada elemento en columnas a la derecha del icono y la etiqueta. Windows El Explorador usa un control de vista de lista para mostrar archivos.

El nombre de clase de ventana para un control de vista de lista es WC LISTVIEW, que se define como \_ "SysListView" en Commctrl.h.

## <a name="notes"></a>Notas

-   Microsoft Active Accessibility no expone texto de elemento de vista de lista en aplicaciones que usan una función de devolución de llamada para agregar el texto al elemento.
-   Los elementos de vista de lista recortados no tienen establecido [**STATE \_ SYSTEM \_ INVISIBLE.**](object-state-constants.md) Para determinar si se recorta un elemento de lista, los clientes pueden comparar las coordenadas del elemento devueltas por [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) con la región de recorte de la ventana.
-   Microsoft Active Accessibility no admite el uso del método [**accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) para navegar al control de encabezado dentro de un control de vista de lista que se encuentra en la vista Detalles. Sin embargo, los clientes usan [**la función AccessibleObjectFromPoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) para llegar al control de encabezado.
-   En el caso de los objetos no rectangulares, como los elementos de vista de lista de la vista Detalles, las coordenadas del rectángulo delimitador del elemento recuperado por [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) producirán un error si se prueban con [**accHitTest**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) porque **accHitTest** determina los límites del objeto píxel a píxel.
