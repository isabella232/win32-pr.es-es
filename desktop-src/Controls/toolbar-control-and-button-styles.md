# <a name="toolbar-control-and-button-styles"></a>Estilos de control y botón de la barra de herramientas

Los siguientes estilos de ventana son específicos de las barras de herramientas. Se combinan con otros estilos de ventana cuando se crea la barra de herramientas.

**Nota** Para Common Controls [versión 6.00](common-control-versions.md), si se usa un estilo [visual](themes-overview.md) con la barra de herramientas, los botones siempre son transparentes independientemente de la configuración de estilo. De lo contrario, el comportamiento de transparencia es normal, como se indica mediante el uso del estilo TBSTYLE \_ FLAT o TBSTYLE \_ TRANSPARENT.

> [!Note]  
> Comctl32.dll versión 6 no es redistribuible, pero se incluye en Windows. Para usar Comctl32.dll versión 6, es especificarlo en un manifiesto. Para obtener más información sobre los manifiestos, vea [Habilitar estilos visuales.](cookbook-overview.md)

**Nota:** Un botón de la barra de herramientas puede tener una combinación de los estilos siguientes. Para especificar un estilo de botón, establezca las marcas adecuadas en el **miembro fsStyle** de la [**estructura TBBUTTON del**](/windows/desktop/api/Commctrl/ns-commctrl-tbbutton) botón. No todos los estilos se pueden combinar.

En shell versión 4.72 y anteriores, las marcas de estilo de ventana y botón tenían el formato TBSTYLE \_ XXX. Si va a compilar una aplicación con la [versión 4.72](common-control-versions.md) o anterior de Commctrl.h, debe usar las marcas de estilo de botón \_ TBSTYLE XXX. Para la versión 5.80 y posteriores, todos los estilos de botón tienen el formato BTNS \_ XXX. Todos los valores DE \_ TBSTYLE XXX tienen valores XXX de BTNS \_ equivalentes, con el mismo significado y valor numérico. Para mayor comodidad, ambos formularios se incluyen en la tabla siguiente.
