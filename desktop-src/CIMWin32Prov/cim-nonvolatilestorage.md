# <a name="cim_nonvolatilestorage-class"></a>Cim \_ NonVolatileStorage (clase)

La **clase CIM \_ NonVolatileStorage** representa las funcionalidades y la administración del almacenamiento no volátil. La memoria no volátil incluye de forma nativa almacenamiento flash y ROM. Además, la memoria no volátil se puede basar en el almacenamiento volátil, si la memoria volátil está basada en una batería. Por ejemplo, este escenario se describiría mediante una instancia de la relación [**\_ AssociatedBattery**](cim-associatedbattery.md) de CIM que hace referencia al almacenamiento no volátil como **dependiente** y la  batería como **antecedente;** y una instancia de la relación [**\_ Cim BasedOn**](cim-basedon.md) que hace referencia al almacenamiento no volátil como el almacenamiento dependiente y el almacenamiento volátil como **antecedente.**

> [!IMPORTANT]
> Las clases CIM (Modelo de información común) DMTF (Distributed Management Task Force) son las clases primarias en las que se han creado las clases WMI. WMI admite actualmente solo los esquemas [de versión CIM 2.x](https://dmtf.org/standards/cim/schemas).

 

La siguiente sintaxis se simplifica desde el código de Managed Object Format (MOF) e incluye todas las propiedades heredadas. Las propiedades se enumeran en orden alfabético, no en orden MOF.
