# <a name="d1201-pushaxisalignedclip-with-antialiased-multisampled-target"></a>D1201: PushAxisAlignedClip con destino multimuestreo suavizado de contorno

PushAxisAlignedClip se usó con un destino multimuestreo. Todavía se emitirá la llamada, pero pueden surgir artefactos si los límites del rectángulo de recorte no están alineados con píxeles. Además, es posible que el resultado representado no represente el patrón multimuestreo esperado.






 

## <a name="possible-causes"></a>Causas posibles

La aplicación interoperaba una textura direct3D multimuestreo como destino de Direct2D y se llamaba PushAxisAlignedClip en el destino.

## <a name="possible-fixes"></a>Posibles correcciones

Evite interoperar texturas multimuestreo si la funcionalidad de recorte es importante. El uso de capas o clips con estos tipos de recursos puede provocar una penalización del rendimiento o resultados de representación inesperados. Esto es especialmente cierto si los límites del clip no están alineados con píxeles. Si es posible, use un medio alternativo para enmascarar el contenido en lugar de un clip, como una máscara de Direct3D. O bien, use un elemento intermedio en lugar de un recurso multimuestreo. Si el recurso multimuestreo está pensado para el suavizado de alias de contenido 2D, considere si el suavizado de contorno integrado por primitivo de Direct2D es suficiente.
