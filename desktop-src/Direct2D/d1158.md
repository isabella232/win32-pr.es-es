# <a name="d1158-pixels-unit-mode-unsupported-with-print"></a>D1158: Modo de unidad de píxeles no compatible con la impresión

Impresión de Direct2D: el comando SetUnitMode(PIXELS) no admitido estaba presente en la lista de comandos enviados para impresión.





 

## <a name="possible-causes"></a>Causas posibles

Direct2D Print no permite un modo de unidad de PÍXELes. El modo de unidad predeterminado es DIPS.

## <a name="possible-fixes"></a>Posibles correcciones

Para solucionar este mensaje, evite establecer el modo de unidad del contexto del dispositivo en PÍXELes al crear contenido imprimible. Si es necesario aplicar una transformación de escala global que afecte a todas las operaciones de dibujo primitivo, considere si una llamada equivalente a ID2D1RenderTarget::SetDpi o ID2D1RenderTarget::SetTransform es suficiente.
