# <a name="d1181-enddraw-incorrect-state"></a>D1181: Estado incorrecto de EndDraw

Se llamó a EndDraw, pero rendertarget no estaba en un estado válido. Esto puede deberse a una llamada a EndDraw sin un BeginDraw correspondiente.






 

## <a name="possible-causes"></a>Causas posibles

La aplicación realizó varias llamadas a EndDraw sin llamar a BeginDraw en el mismo destino. Al usar Direct2D, las aplicaciones deben hacer coincidir cada llamada a EndDraw con una llamada anterior a BeginDraw. Como alternativa, este mensaje puede producirse si el destino está en un estado erróneo y no se puede representar.

## <a name="possible-fixes"></a>Posibles correcciones

Asegúrese de que cada llamada a BeginDraw coincide con un EndDraw posterior. Al enviar comandos a un destino mediante Flush o EndDraw, compruebe que el resultado devuelto es S OK antes de intentar usar de nuevo \_ el destino.
