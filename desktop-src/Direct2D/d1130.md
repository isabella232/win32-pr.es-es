# <a name="d1130-dxgi-format-disallowed-by-caps"></a>D1130: formato de DXGI no permitido por Cap

El dispositivo no tiene la capacidad de admitir el formato de DXGI dado.






 

## <a name="possible-causes"></a>Causas posibles

En el caso de las operaciones de Direct2D que se ejecutan en hardware y utilizan un formato de superficie, el formato de superficie debe ser compatible con el nivel de característica del hardware. Por ejemplo, si el hardware es el nivel de característica 9, puede que no admita formatos de 16 BPP o formatos SRGB, como el formato de DXGI \_ \_ B8G8R8A8 \_ UNORM \_ sRGB.

## <a name="possible-fixes"></a>Posibles correcciones

Vea [niveles de características de Direct3D](/previous-versions/windows/apps/hh994923(v=win.10)) para obtener instrucciones sobre las diferencias entre los niveles de características y los formatos admitidos. Considere la posibilidad de usar el [**método ID3D11Device:: CheckFormatSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkformatsupport) para comprobar que el formato requerido sea compatible con el hardware antes de usarlo con Direct2D.
