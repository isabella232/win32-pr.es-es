# <a name="handles-and-objects"></a>Identificadores y objetos

Un *objeto* es una estructura de datos que representa un recurso del sistema, como un archivo, un subproceso o una imagen gráfica. Una aplicación no puede tener acceso directamente a los datos de objeto o al recurso del sistema que representa un objeto. En su lugar, una aplicación debe obtener un *identificador* de objeto, que puede usar para examinar o modificar el recurso del sistema. Cada controlador tiene una entrada en una tabla mantenida internamente. Estas entradas contienen las direcciones de los recursos y los medios para identificar el tipo de recurso.
