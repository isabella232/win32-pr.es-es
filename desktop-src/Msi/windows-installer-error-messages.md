# <a name="windows-installer-error-messages-for-developers"></a>Windows Mensajes de error del instalador (para desarrolladores)

> [!NOTE]  
> Este artículo está destinado a desarrolladores de software que usan Windows Installer para compilar paquetes de instalador para sus aplicaciones. Si es un usuario que experimenta dificultades con el equipo durante o después de instalar o desinstalar una aplicación, póngase en contacto con el servicio de atención al cliente para el software que está intentando instalar o quitar. Si cree que necesita soporte técnico para un producto de Microsoft, vaya a nuestro sitio de soporte técnico en [support.microsoft.com](https://support.microsoft.com).

El instalador de Windows devuelve los códigos de error detallados en este tema y tienen códigos de error de 1000 o superiores. Los códigos de error numerados de 1000 a 1999 son errores de envío y deben crearse en la [tabla Error](error-table.md). Los códigos de error numerados mayores que 2000 son errores internos y no tienen cadenas de creación, pero pueden producirse si el paquete de instalación se ha escrito incorrectamente. Para obtener códigos de error específicos  de las funciones Windows InstallerMsiExec.exey **InstMsi.exe**, veaMsiExec.exe y [InstMsi.exe mensajes de error](error-codes.md). Para obtener una lista de códigos de error reservados, vea [Tabla de errores](error-table.md). Puede buscar soluciones en Internet o en el sitio de [soporte técnico de Microsoft](https://support.microsoft.com/) para muchos de los mensajes de la tabla siguiente.

Los desarrolladores de paquetes de instalación también pueden probar la coherencia interna de sus paquetes mediante evaluadores de coherencia internos. Para obtener más información, vea [Internal Consistency Evaluators - ICEs (Evaluadores](internal-consistency-evaluators-ices.md)de coherencia internos: TIC).
