<dl> <dt>

<span id="_msi_cabinet_file_using_windows_installer_gly"></span><span id="_MSI_CABINET_FILE_USING_WINDOWS_INSTALLER_GLY"></span>**archivo de gabinete**
</dt> <dd>

Archivo único, normalmente con una .cab, que almacena archivos comprimidos en una biblioteca de archivos. El formato de gabinete es una manera eficaz de empaquetar varios archivos porque la compresión se realiza a través de los límites de los archivos, lo que mejora significativamente la relación de compresión. Para obtener información sobre cómo crear gabinetes, vea [Archivos de gabinete](cabinet-files.md).

</dd> <dt>

<span id="_msi_checksum_gly"></span><span id="_MSI_CHECKSUM_GLY"></span>**Checksum**
</dt> <dd>

Valor calculado que depende del contenido de un archivo. Se almacena con datos para detectar daños en los archivos. El sistema comprueba este valor para asegurarse de que los datos no se han corrugado.

</dd> <dt>

<span id="_msi_component_using_windows_installer_gly"></span><span id="_MSI_COMPONENT_USING_WINDOWS_INSTALLER_GLY"></span>**Componente**
</dt> <dd>

Parte más pequeña de una instalación que controla el instalador, también un bloque de creación de una aplicación o característica desde la perspectiva del programador. Algunos ejemplos de componentes son un grupo de archivos relacionados, un objeto COM o una biblioteca. Para obtener más información, [vea Componentes y características](components-and-features.md).

</dd> <dt>

<span id="_msi_committing_databases_using_windows_installer_gly"></span><span id="_MSI_COMMITTING_DATABASES_USING_WINDOWS_INSTALLER_GLY"></span>**confirmar bases de datos**
</dt> <dd>

Cambios acumulados realizados en una base de datos. Los cambios no se reflejan en la base de datos real hasta que se confirma la base de datos. Para obtener más información, vea [Confirmar bases de datos](committing-databases.md).

</dd> <dt>

<span id="_msi_controlevent_gly"></span><span id="_MSI_CONTROLEVENT_GLY"></span>**ControlEvent**
</dt> <dd>

Aspecto de la funcionalidad de la interfaz de usuario del instalador. Un control ControlEvent típico desencadena alguna acción por parte del instalador tras la activación de un control de cuadro de diálogo u otro evento. Para obtener más información, vea [Información general de ControlEvent.](controlevent-overview.md)

</dd> <dt>

<span id="_msi_costing_gly"></span><span id="_MSI_COSTING_GLY"></span>**Costando**
</dt> <dd>

Método utilizado por el instalador para determinar los requisitos de espacio en disco para la instalación. El costo tiene en cuenta factores como si es necesario sobrescribir los archivos existentes. Para obtener más información, vea [File Costing](file-costing.md).

</dd> <dt>

<span id="_msi_.cub_file_gly"></span><span id="_MSI_.CUB_FILE_GLY"></span>**Archivo .uu.**
</dt> <dd>

Módulo de validación que almacena y proporciona acceso a acciones personalizadas de ICE. Para obtener más información, vea [Archivo .uu. de ejemplo.](sample--cub-file.md) Para obtener más información, vea [también Windows Installer File Extensions](windows-installer-file-extensions.md).

</dd> <dt>

<span id="_msi_custom_action_using_windows_installer_gly"></span><span id="_MSI_CUSTOM_ACTION_USING_WINDOWS_INSTALLER_GLY"></span>**acción personalizada**
</dt> <dd>

Escrito por el autor del [*paquete,*](p-gly.md) pero no integrado en el instalador como una acción estándar. Para obtener más información, vea [Acciones personalizadas.](custom-actions.md)

</dd> </dl>
