# <a name="determining-shim-status"></a>Determinar el estado de corrección de compatibilidad (shim)

## <a name="platforms"></a>Plataformas

<dl> Clientes: Windows 8  
Servidores: Windows Server 2012  
</dl>

## <a name="description"></a>Descripción

Windows 8 eventos cuando las aplicaciones se están recortando por diversos motivos. La manera más fácil de determinar si la aplicación se está recortando es comprobar el visor de eventos. Vaya a Applications and Services Logs Microsoft Windows Application-Experience Program Telemetry (Telemetría \\ del programa microsoft Windows Application-Experience servicios y \\ aplicaciones).

## <a name="mitigation"></a>Mitigación

La mejor manera de salir de la corrección de compatibilidad es cambiar el nombre del archivo ejecutable o aumentar la versión principal en 1 (volver a compilar el archivo ejecutable).
