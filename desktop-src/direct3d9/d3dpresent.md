# <a name="d3dpresent"></a>D3DPRESENT

Describe la relación entre la frecuencia de actualización del adaptador y la velocidad a la que se [**completan**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3dswapchain9-present) las [**operaciones**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3dswapchain9-present) Present o Present. Estos valores también sirven como valores de marca para el campo PresentationIntervals de [**D3DCAPS9.**](/windows/desktop/api/D3D9Caps/ns-d3d9caps-d3dcaps9)

## <a name="remarks"></a>Observaciones

El modo de ventana admite D3DPRESENT \_ INTERVAL \_ DEFAULT, D3DPRESENT \_ INTERVAL IMMEDIATE y \_ D3DPRESENT \_ INTERVAL \_ ONE. D3DPRESENT \_ INTERVAL DEFAULT y D3DPRESENT INTERVAL ONE son casi equivalentes (consulte la información sobre la resolución \_ del temporizador a \_ \_ continuación). Se realizan de forma similar a COPY VSYNC, ya que solo hay un presente por fotograma y evitan la lágrima con el \_ haz siguiente. Por el contrario, D3DPRESENT \_ INTERVAL IMMEDIATE intentará proporcionar una tasa de presentación \_ ilimitada.

El modo de pantalla completa admite un uso similar al modo de ventana al admitir D3DPRESENT INTERVAL IMMEDIATE independientemente de la frecuencia de actualización o \_ \_ el efecto de intercambio. D3DPRESENT INTERVAL DEFAULT usa la resolución predeterminada del temporizador del sistema, mientras que D3DPRESENT INTERVAL ONE llama a \_ \_ \_ \_ [**timeBeginPeriod**](/windows/win32/api/timeapi/nf-timeapi-timebeginperiod) para mejorar la resolución del temporizador del sistema. Esto mejora la calidad de la sincronización vertical, pero consume un poco más de tiempo de procesamiento. Ambos parámetros intentan sincronizarse verticalmente.
